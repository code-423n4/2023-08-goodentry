This allows handling user assets without having to give allowance to position managers It also allows easy deleveraging (since can move tokens while there is ongoing debt) In exchange the PM needs to be secure and return all assets else there could be bad debt. Or another user's assets could be stolen. So transfers are only allowed if coming from the user moving its own funds (for leveraging/deleveraging purposes, or recreating TR asset), or if the health factor is low and (soft) liquidation is allowed (edited)
